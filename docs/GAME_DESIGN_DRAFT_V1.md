# Anchor 简易策划案 V1

## 总览

**游戏名：**

**背景：**船因为 xx 原因抛锚在海里一个地方，但海域似乎不对劲。

**立意：**理智是现实的锚点，船锚是地理的锚点，没了人就迷失了。

**游戏类型：**恐怖游戏，类似《玩具熊的午夜后宫》，但更类似独立游戏 *These Darker Tides*，同时移动更多（需要穿越房间），第一人称视角。

**具体玩法：**

在一张较小的地图（抛锚的船）内，分布着房间和走廊，怪物从海里侵入到船进行破坏。玩家需要根据不同怪物的应对方法驱赶/消灭怪物，保证 san 值不归零的情况下，将卡住的锚修好以离开险境，保住锚的血条直至天亮或者游戏结束。

**终极目标：**保证 san 值不为零，修好锚。

**核心循环：**修理锚的回收装置 -> 怪物攻击 -> 应对攻击 -> 修理锚。

**玩家行为：**

- 阶段 1：修理回收装置 -> 怪物破坏窗 -> san 值少量减少 -> 查看监控发现怪物位置 -> 玩家前往修窗 -> 怪物离开 -> 修理回收装置。
- 阶段 2：修理回收装置 -> 怪物尝试熄灭蜡烛 -> san 值大量减少 -> 查看监控发现怪物位置 -> 玩家前往驱赶 -> 怪物离开 -> 修理回收装置。

## 详细设定

### 入侵

即怪物不同阶段。

- 第一阶段：怪物在窗外，玩家在屋内，怪物尝试破坏窗户进入房间。
- 第二阶段：怪物与玩家在屋内，怪物尝试通过熄灭蜡烛让玩家 san 值归 0。

### 怪物

- 背景设定：海里出来的，害怕光亮，间接伤害玩家 san 值。

### 不同阶段怪物攻击模式与应对方式

#### 窗外（入侵前，阶段 1）

- 阶段 1 进入条件：进入游戏。
- 攻击：每隔随机一段时间，攻击窗户，使窗户耐久度降低（不需要完整模型出现）。持续时间 = (8,15]s。
- 攻击时指引：攻击位置有点声源播放撞击音效，玩家根据声音大小判断与怪物距离远近；窗口光源变红。
- 玩家应对：使用房间内木板，读条将木板钉上窗户，从而修补窗户耐久度。
- 攻击过程中 san 值变化，公式见下面的“玩家 san 值”。
- 耐久归零后：san 值最大值变为 80。
- 耐久归零无法修复。

#### 房间内（入侵后，阶段 2）

- 阶段 2 进入条件：任一窗户耐久度 = 0。
- 攻击：伸出黑手掐灭蜡烛（仅需黑手模型），黑手会持续浮空 + 慢慢靠近的动画，最多 15s 后掐灭蜡烛。
- 攻击时指引：出现全局的呓语音效，正在被熄灭的蜡烛上方出现怪物手，出现持续时间 = [5, 8]S。
- 玩家应对：前往对应房间，点击驱逐黑手/重燃蜡烛。
- 蜡烛熄灭后，san 值变化，公式见下面的“玩家 san 值”。

### 玩家 san 值

- san 值 = 血量，san 值（max）= 100，san 值归零结束游戏。
- san 值变化公式：
  - （阶段 1&2）全部蜡烛亮着：san 值 = current + delta（time）* 1.5。
  - （阶段 1）窗户被攻击：san 值 = current - delta（time）* 1。
  - （阶段 2）有任一蜡烛熄灭：san 值 = current - delta（time）* 5 * 熄灭的蜡烛个数。

## 场景

- 背景：船舱。
- 场景道具：
  - 窗户：每个房间 2 个。
	- 耐久度（int），max = 100。
	- 状态：
	  - 正常。
	  - 正被怪物袭击：耐久度 = current - 5 * delta（time）。
	  - if 全局游戏时间 > 2min：耐久度 = current - 8 * delta（time）。
	  - 修补：耐久度 = current + 4 * delta（time）。
  - 蜡烛：每个房间 2 个，共 4 个。
	- 单个状态：是否亮（bool）。
  - 木板：长按窗户，木板一片片自动铺上去。
  - 锚的回收装置：长按修理，玩家累计修理时间达到 2min 即可完成修复。

## 交互设计

**玩家状态：**理智。

**玩家交互行为：**

1. 修锚的装置：在船头房间长按控制台读条。
2. 修窗：长按窗户，窗户正下方显示进度条。
3. 点蜡烛：长按读条，时间相当较短。
4. 驱赶黑手：连点黑手，模型抽搐，一定次数后消失。
5. 玩家移动：在屏幕右下角做个小地图，显示两个房间，玩家点击标示房间按钮进行移动（形式想是直接黑屏切换）。
