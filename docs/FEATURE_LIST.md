# Anchor 功能点列表

> 来源：[腾讯文档 - 功能点列表](https://docs.qq.com/sheet/DU0xXZ1dXRmp5Z1R5?tab=000001)

| 序号 | 模块 | 功能点 | 描述 |
|---:|---|---|---|
| 1 | 基本操作 | 玩家移动 | 第一人称，固定视角，在固定位置上镜头可以跟随玩家鼠标上下左右轻微转动。换房间的时候镜头平滑移动。 |
| 2 | 物品交互 | 玩家修理锚回收装置 | 长按鼠标左键读条维修。<br>玩家与其累计交互时间达到 2min 则完成修理，游戏挑战成功。 |
| 3 | 物品交互 | 玩家交互前置 | 先在固定点位鼠标左键点击一次回收装置/蜡烛/窗户进入正面面对物品的特写，才可进行交互。<br>鼠标右键单机退出特写。 |
| 4 | UI | 修理回收装置进度显示 | 常驻，显示在屏幕正下方。 |
| 5 | 玩家数值 | 玩家 san 值变化 | （阶段 1&2）全部蜡烛亮着：san 值 = current + delta（time）* 1<br>（阶段 1）窗户被攻击：san 值 = current - delta（time）* 2<br>（阶段 2）有任一蜡烛熄灭：san 值 = current - delta（time）* 4 * 熄灭的蜡烛个数<br>san 值归零结束游戏。<br>san 值 < 20 时播放音效，屏幕变红。 |
| 6 | UI | san 值显示 | 常驻，显示在屏幕左上角。 |
| 7 | 怪物攻击 | 怪物攻击窗户 | 怪物每隔随机时间间隔，就会攻击任一窗户使其耐久降低。<br><br>耐久度（int），max = 100，delta_time = 2s；<br>正被怪物袭击：耐久度 = current - 5 * delta_time；<br>if 全局游戏时间 > 2min：耐久度 = current - 8 * delta_time；<br>修补：耐久度 = current + 4 * delta（time）<br>攻击持续时间：(8,15]S；<br>任一窗户耐久度归零后，窗户禁用，玩家 san 值上限变为 80，进入阶段 2。 |
| 8 | 怪物攻击场景变化 | 被攻击窗户指示 | 怪物攻击时，被攻击的窗户有点声源（距离越远声音越小，反之越大）播放撞击音效，并且对应窗户出现红光，怪物离开后红光与音效消失。 |
| 9 | 物品交互 | 玩家修补窗户 | 长按鼠标左键 2S 读条，使其耐久恢复。<br>怪物是否正在攻击不影响是否进行修补。 |
| 10 | UI | 窗户耐久显示 | 鼠标悬浮在窗户上显示耐久度，显示在窗户正下方。 |
| 11 | 怪物攻击 | 怪物攻击烛火 | 黑手随机出现在任一蜡烛上方，如未在其消失前发现并驱赶，则蜡烛熄灭。<br>黑手出现持续时间：[5, 8]S，delta_time_1 = 1s<br>有任一蜡烛熄灭：san 值 = current - delta_time_1 * 5 |
| 12 | 物品交互 | 玩家驱赶黑手 | 连点黑手，黑手模型快速上下抽出，连点 5 次后消失。 |
| 13 | 物品交互 | 玩家重燃蜡烛 | 长按鼠标左键 1S 读条，重燃。<br>怪物是否正在攻击不影响是否进行修补。 |
| 14 | 怪物攻击场景变化 | 被攻击蜡烛指示 | 怪物攻击时，播放全局呓语音效，被赶走/蜡烛熄灭后消失。<br>攻击时怪物手出现在蜡烛上方，并播放动画。 |
| 15 | UI | 退出窗口 | 入口：常驻，位置在右上角。<br>功能：点击后弹出窗口二次确认“是否退出游戏？”，下设一个按钮“确认”，点击确认后退出游戏。 |
| 16 | UI | GG 界面 | 玩家完成维修：黑屏，中央白色像素大号文字（需要无版权字体）显示：“平安启航”。下面白色小字显示“锚一归位，我就恢复了平静。我筋疲力尽，扑倒在小床上。想必我睡着了，因为醒来时满脸映着星光。”<br><br>玩家 san 值归零：黑屏，中央红色像素大号文字（需要无版权字体）显示：“坠海”。下面白色小字显示“海水涌入我的咽喉，这座现实的地狱，终归不是人的王国。”<br><br>下设按钮“返回主界面”，点击后返回主页面。 |
| 17 | UI | 主界面 | 点击“开始游戏”开始。 |
